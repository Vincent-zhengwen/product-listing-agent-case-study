# Agent 核心能力

这个案例展示的是一个面向商品上架的 Agent: 用户给出货源链接, Agent 产出一套可发布的商品内容资产。货源也可以来自关键词检索工具, 但本案例使用手动提供的义乌购链接, 让输入和输出更容易复核。

## 1. 货源理解

Agent 首先读取货源页中的标题、属性、图片和规格信息, 再区分哪些内容适合进入 C 端商品页, 哪些只是供货语境。

在桌布案例中, 原始页面同时出现了桌布材质、尺寸、款式, 也出现了代理、代发、跨境采购、开票等 B2B 信息。Agent 的目标不是复制货源页, 而是保留商品事实, 移除不适合出现在买家页里的供货信息。

## 2. 事实约束

商品标题、属性和图片文案必须来自可验证事实。材质、规格、颜色、形状、适用场景等字段不能凭空补全; 不确定的信息以空缺或保守表达处理。

这个约束让 Agent 不只是“写得像商品页”, 而是把上架内容控制在可发布范围内。

## 3. 类目化策略

不同类目的购买决策不同。桌布关注铺上后的餐桌效果、面料纹理、垂坠边缘、尺寸选择和浅底确认; 灯具、香薰、收纳、餐具则需要不同的主图任务和详情节奏。

因此图片规划不是固定模板, 而是先识别商品类目, 再为每张主图和详情屏分配明确任务。

## 4. 文案与属性生成

Agent 将货源事实转成商品发布需要的结构化内容:

- 标题: 控制平台长度, 避免极限词、批发词和不相关关键词。
- 属性: 保留材质、颜色、尺寸、风格等可确认字段。
- 卖点: 用商品特征支撑买家利益, 避免泛化口号。
- 服务信息: 低权重展示, 不喧宾夺主。

## 5. 图片资产生成

Agent 为商品页输出完整图片资产:

- 5 张 800x800 主图。
- 8 屏 750 宽详情图。
- 1 张拼接后的详情长图。

桌布案例中的主图分别承担铺桌首图、尺寸规格、面料纹理、边缘垂坠和浅底确认任务; 详情图再补充场景、规格和服务信息。

## 6. 交付检查

长链路 Agent 容易在某个环节出现内容缺失、事实漂移或图片角色混乱。因此交付前需要检查:

- 主图数量和尺寸是否达标。
- 详情图是否完整拼接。
- 标题和属性是否符合事实约束。
- 图片文案是否出现不适合公开展示的生产过程词。
- 详情页是否出现空白、裁切或模板感过重的问题。

这个案例的重点不是单次生成一张好看的图, 而是让 Agent 形成从货源理解到完整上架交付的闭环。

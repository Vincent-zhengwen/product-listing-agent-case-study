# 上架 Agent 公开版 Playbook

这份 playbook 是脱敏后的展示版本, 用来说明 Agent SDK 阶段如何把业务经验写成可执行规则。它不包含生产凭据、抓取细节、平台账号信息或内部调试流程。

## 目标

把一个 B2B 货源链接转成可复核的商品上架资产:

- 淘宝风格标题。
- 可验证商品属性。
- 5 张主图。
- 8 屏详情图。
- 1 张详情长图。
- 交付前质量检查结果。

## 执行顺序

1. 读取货源事实: 标题、属性、规格、源图和供货语境。
2. 判断商品主体: 明确真正出售的商品, 把道具和供货信息降权。
3. 选择类目策略: 不同品类使用不同主图任务和详情节奏。
4. 生成计划: 输出标题、属性、卖点、主图任务和详情图任务。
5. 事实校验: 检查材质、尺寸、认证、功能和买家可见文案。
6. 生成资产: 按计划生成主图、详情图和详情长图。
7. 交付检查: 检查数量、文件、空白、裁切、内部词和 B2B 词。
8. 修正后交付: 有 blocker 就回到计划或资产生成环节。

## 桌布类目策略

桌布不是“装饰图片合集”, 主图和详情图要围绕购买决策展开:

| 图片位置 | 任务 |
|---|---|
| 主图 1 | 铺桌首图, 让买家看到铺上后的餐桌效果 |
| 主图 2 | 尺寸规格, 帮买家确认是否适合自家餐桌 |
| 主图 3 | 面料纹理, 展示棉麻质感 |
| 主图 4 | 边缘垂坠, 展示桌沿效果 |
| 主图 5 | 浅底确认, 方便确认花型和主体 |

详情图前半段展示场景和卖点, 后半段补充规格、款式和低权重服务信息。

## 事实边界

买家可见内容只能使用货源中可确认的信息。以下内容不能凭空生成:

- 材质、尺寸、重量、容量、功率、电压。
- 认证、检测、产地、包装数量。
- 防水、防油、防霉、食品级等功能或安全承诺。
- 销量、评价、排名、价格优势。

不确定的信息宁可空缺, 不要补猜。

## 禁止泄漏的语境

买家页不应该出现这些信息:

- 代理、批发、一件代发、MOQ、OEM、厂家直销。
- 生成、模型、提示词、推理、demo、preview。
- 主图编号、详情编号、内部策略、诊断原因。

## 交付门槛

一轮生成完成后, Agent 需要确认:

- 主图正好 5 张。
- 详情图至少 8 屏。
- 详情长图存在。
- 标题和属性没有未证实事实。
- 图片文案没有内部过程词。
- 最终资产可以被打开、复核和展示。

商品上架 Agent 架构演进

从 Workflow 到单 Agent + 多工具，再到多专职 Agent 工作流和 Claude Agent SDK，展示一个上架 Agent 如何把货源理解、工具调用、职责拆分和产物验证逐步纳入架构。

四轮架构，解决不同阶段的上架生成问题

阶段一 代表 case：花瓶案例 动态判断不足

Workflow：固定步骤流程

阶段定位：用固定步骤把货源标题、属性、图片转成上架草稿，先跑通从 B2B 货源到 C 端商品素材的基础链路。

这一阶段的架构

货源标题 / 属性 / 图片

将源商品的基础信息作为固定流程输入。

VLM 看图

识别商品外观、材质和可用于表达的基础视觉信息。

固定模板生成

按顺序生成标题、卖点、属性、主图和详情图。

合规检查 / 草稿

完成基础检查后输出 Listing 草稿。

运行瓶颈

瓶颈描述 固定流程能保证步骤执行，但缺少对货源素材差异的动态判断。
发生环节 货源输入 -> 固定生成链路 -> 模板化产出。
系统边界 Workflow 负责稳定执行预设步骤，但不负责重新判断每张素材该承担什么信息任务。
下一阶段架构升级

引入单 Agent + 多工具，让 Agent 根据货源情况主动调用浏览、VLM、搜索、渲染和检查工具。

阶段二 代表 case：桌布案例 职责过载

单 Agent + 多工具：单 Agent 工具调度

阶段定位：一个 Agent 统一调度浏览、VLM、竞品搜索、生图、渲染和合规检查工具，开始具备动态处理不同货源的能力。

这一阶段的架构

货源页

Agent 从页面和素材中读取商品上下文。

单 Agent

统一做商品理解、策略判断、文案和图片规划。

多工具调用

浏览、VLM、竞品搜索、生图、渲染和合规检查接入同一循环。

上架素材草稿

输出图文素材，但最终收口仍依赖单 Agent 的整体判断。

运行瓶颈

瓶颈描述 工具能力集中到一个 Agent 后，商品理解、策略、文案、属性、图片规划和质检都压在同一个决策循环里，长链路交付容易不稳定。
发生环节 单 Agent 决策 -> 多工具调用 -> 最终交付包。
系统边界 工具调用能力已经进入链路，但工具调用成功不等于上架素材稳定交付。
下一阶段架构升级

拆分专职 Agent，让商品理解、策略、文案、属性、图片规划和质检进入不同阶段合同。

主图组

单 Agent 调用工具后产出的 5 张主图。

详情图组

单 Agent 工具链产出的 8 张详情图。

阶段三 代表 case：小夜灯案例 最终产物验证

多专职 Agent 工作流：专职 Agent 流水线

阶段定位：将商品理解、竞品分析、关键词、策略、文案、属性、图片规划和质检拆成专职 Agent，形成结构化上架流水线。

这一阶段的架构

ProductVLMAgent

抽取商品事实、场景、材质、尺寸和人群。

Competitor / Keyword

补充竞品信息、关键词和平台表达方向。

Strategy / Copy / Attribute / ImagePlanner

生成策略、文案、属性和图片规划。

JudgeAgent

检查阶段产物并输出上架素材包。

运行瓶颈

瓶颈描述 分工让规划和字段更稳定，但最终商品图和素材文件本身仍需要更强的产物级检查与修正。
发生环节 专职 Agent 输出 -> JudgeAgent 评分 -> 最终 artifact。
系统边界 多专职 Agent 解决职责分工，但还没有把最终产物验证变成稳定执行闭环。
下一阶段架构升级

把源数据、工具结果、最终图片、质量检查和修正动作放进同一个执行闭环。

多专职 Agent 规划后产出的 8 张详情图。

阶段四 代表 case：桌布交付案例 生成 / 检查 / 修正

Claude Agent SDK：长任务执行闭环

阶段定位：将上架规则沉淀成 skill/playbook，由 Claude Agent SDK 在文件上下文和工具上下文中完成生成、检查和修正。

这一阶段的架构

Skill / Playbook

沉淀上架规则、图片标准和质量检查经验。

Claude Agent SDK 执行循环

在文件上下文和工具上下文中持续推进任务。

商品图生成 / 质量检查

围绕最终商品图和素材结果做检查。

修正 / 素材交付包

根据检查结果修正并形成上架素材交付包。

运行结果

素材内容 交付包包含标题、属性、5 张主图、8 屏详情图和详情长图。
检查对象 源数据 / 工具结果 / 最终图片 / 质量检查 / 修正动作需要进入同一个执行循环。
系统边界 面向最终商品图和上架素材包进行验证与修正，而不是只检查中间计划或字段。
最终架构形态

以 Claude Agent SDK 执行循环承接长任务状态、文件上下文、工具调用、质量检查和自我修正。

四阶段架构对照

阶段 架构形态 主要瓶颈
Workflow 固定步骤流程 缺少动态判断
单 Agent + 多工具 单 Agent 工具调度 决策职责过载
多专职 Agent 工作流 专职 Agent 流水线 最终产物验证不足
Claude Agent SDK 长任务执行闭环 面向商品图和素材包稳定交付
